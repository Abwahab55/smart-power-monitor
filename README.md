## Overview

This project implements a full-stack **Industrial IoT power monitoring system** on AWS. A simulated edge device (Raspberry Pi / MCU) reads power sensor data — voltage, current, active power, power factor, frequency, and temperature — and publishes it every 5 seconds via **MQTT over TLS** to **AWS IoT Core**.

A serverless backend processes each reading in real-time: detecting faults, persisting data to **DynamoDB**, archiving raw payloads to **S3**, and firing **SNS email/SMS alerts** when anomalies are detected. A **REST API** backed by API Gateway and Lambda serves the data to a live web dashboard.

**Why this project?** It directly connects power electronics expertise (SiC systems, power converters, circuit analysis) with cloud-native architecture — a strong portfolio piece for cloud IoT and edge computing roles.

## Architecture

```
Edge Device (Raspberry Pi / MCU)
        │  MQTT over TLS
        ▼
  AWS IoT Core ──── IoT Rule ────► Lambda (IoT Processor)
                                        │
                    ┌───────────────────┼────────────────┐
                    ▼                   ▼                ▼
               DynamoDB             AWS SNS            S3
           (time-series)        (email alerts)    (raw archive)
                    │
                    ▼
          API Gateway + Lambda (REST API)
                    │
                    ▼
            Web Dashboard (live charts)
```

## AWS Services Used

| Service | Role |
|---|---|
| AWS IoT Core | MQTT broker, device registry, IoT rules engine |
| AWS Lambda | Serverless compute — IoT processor + REST API |
| Amazon DynamoDB | Time-series storage with TTL (30-day retention) |
| Amazon S3 | Raw payload archive, versioned |
| Amazon SNS | Email / SMS anomaly alerts |
| Amazon API Gateway | REST API for dashboard |
| AWS IAM | Roles and policies for secure access |

## Getting Started

### Prerequisites

- Python 3.8+
- AWS account with CLI configured (`aws configure`)
- AWS credentials with IAM, Lambda, DynamoDB, S3, SNS, IoT Core permissions

### 1. Clone the repository

```bash
git clone https://github.com/Abwahab55/smart-power-monitor.git
cd smart-power-monitor
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Linux / macOS
venv\Scripts\activate           # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Download Amazon Root CA certificate

```bash
mkdir -p certs
curl -o certs/AmazonRootCA1.pem \
  https://www.amazontrust.com/repository/AmazonRootCA1.pem
```

### 5. Provision all AWS resources (one command)

```bash
python infrastructure/setup_aws.py \
  --region eu-central-1 \
  --email your@email.com
```

This creates everything automatically:
- DynamoDB table with TTL
- S3 bucket (versioned)
- SNS topic + email subscription
- IAM role with required permissions
- Lambda functions (IoT processor + REST API)
- IoT Thing, certificate, policy
- IoT Core Rule → Lambda trigger
- Saves `config.json` and `certs/` for the device

> Confirm the SNS subscription email before testing alerts.

### 6. Run the device simulator

**AWS mode (publishes to IoT Core):**

```bash
python device/simulator.py \
  --endpoint <your-iot-endpoint>.iot.eu-central-1.amazonaws.com \
  --cert  certs/device-cert.pem \
  --key   certs/private-key.pem \
  --ca    certs/AmazonRootCA1.pem \
  --interval 5
```

**Local mode (no AWS needed — for testing):**

```bash
python device/simulator.py --local
```

## REST API Endpoints

After deployment, query your data via API Gateway:

| Method | Endpoint | Description |
|---|---|---|
| GET | `/readings?device_id=X&limit=100` | Latest N readings |
| GET | `/readings/stats?device_id=X` | Aggregated statistics |
| GET | `/alerts?device_id=X&limit=50` | Recent anomalies only |

**Example response — `/readings/stats`:**

```json
{
  "device_id": "smart-power-monitor-device-01",
  "stats": {
    "reading_count": 500,
```

## Testing Lambda Locally

